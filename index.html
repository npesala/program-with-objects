The Basics of Web and HTML

Web Overview

The basics of web include HTML, URLS, HTTP, web apps, and the web itself. The browser makes requests via the internet to servers. These requests are using http and servers respond with files that your browser displays Ex: firefox, chrome Servers are computers similar to are but they host files and are not placed on desks. Http is the main protocol that the browser uses to communicate with the servers.

HTML

Hyper Text Markup Language It is the basis of almost every webpage and glues everything together. It is made up of text context "what you see" and markup "what it looks like”. The majority of documents on the web are HTML documents.

Tags and Elements

HTML documents are made up of element created by tags. In HTML we tell the browser the type of each element through tags. Ex of tags: b-tag: bold, em-tag: emphasis (italics), img-tag: images, br-tag: break (forces HTML to produce multiple lines),p-tag: paragraph (creates invisible box around text)

Inline and Block

Inline ends the current line. Ex: br-tag Block creates an invisible box around the text. Ex: p-tag

Creating a Structured Document with HTML

HTML, CSS and Developer Tools

HTML controls the structure of the page, while CSS controls style of the page. HTML and CSS are both “languages”. There is also something else called JavaScript which controls the interactive components. To see the structure of a page first you go to file and click tools and then click developer tools to see the structure

Tree-like Structures

DOM (Document Object Model) is the tree-like structure of the page that programmers use. HTML tags become elements and elements form a tree. Each HTML tag creates an element in the DOM that the browser uses to display the page. The tree-like structure shows the relationship between a parents and a child element. The tree-like structure is similar to that of a family tree. One mother can have many children and those children can have children. There can also be boxes inside boxes, inside boxes and so on.

Boxes!

Every element is a box, even the images that are not rectangular shaped. Boxes are defined with div tags.

Sublime Text Editor

There are special types of text editor that help programmer program more easily By guessing what tag you may want to use or when you select a tag, it creates a closing tag automatically.

Adding CSS Style to HTML Structure

CSS

CSS stands for cascading style sheets. The 'C' in CSS is the most important aspect. It is a code written to control the “style” of HTML elements. CSS attributes and values website: https://developer.moxilla.org/en-US/docs/web/css

Code Comments

Code comments are small notes for the developer. Browser will ignore the comments, hence the users will not see them.

Importance of Avoiding Repetition

Repeating the same code can lead to errors when developers want to make a change because the requirements have changed. Developer has to consistently make the same update to the code in all the instances of repeated code, otherwise errors will occur.

By avoiding the repetition of code, you get the benefit of consistent behavior in your web page. In addition, avoiding repetition of code obviously saves time.

Box Model and Box Sizing

The box model includes contents, padding, borders and margins. HTML elements are box and each box has 4 components. Because there are so many components to each box, it can often be hard to get the size of the box right. There are two techniques you can use to help deal with sizing issues; set sizes in terms of percents rather than pixels or set the box-sizing attributes to border-box for every element.

Introduction to Serious Programming

Computers and Computer Programs

A computer is a universal machine that can be programmed to do anything, unlike a toaster for example which would need to be physically altered to perform a different task. The power of the computer is that it can execute steps really fast. We can execute billions of instructions in one second.

A computer program is a sequence of steps that a computer can follow to do something useful. It gives us a way to tell the computer what steps to take. There are many different languages for programming computers. Ex: web browser, python interpreter, python code, calendar app on phone

Programming Language

A programming language is what programmers use to tell a computer what to do. Python is one example of a programming language.

Python

Python is a programming language. Instead of the program running directly on the computer, the programs we write will be an input to the python program which runs the computer. Python comments start with “#”. The python interpreter executes and understands the programs by running them into a language the computer can understand.

Language Ambiguity and Verbosity

New languages were invented to program computers rather than using English, because of ambiguity. Different people can interpret the same sentence to mean many different things. We need to make sure the computer interprets the program the exact same way that we intended for it to be interpreted.

Another reason is that natural languages like English are verbose. To write a program we need to describe exactly what the computer should do in a very precise sequence of steps. If we had to describe all those details using a natural language, it would require a huge amount of text. With python, we would only need a few lines of code to describe something complicated and describe it in a precise, step by step way.

Backus-Naur Form

The purpose of the Backus-Naur form is to be able to precisely describe exactly the language in a way that’s very simple and concise. Basically, it is the replacing of non-terminals with their replacements until we’re left with only terminals.

Grammar

Programming languages have grammar, just like natural languages. A grammar is a specification of what is "correct" and what is "incorrect”, in English. But, computers aren't smart enough to do this. This means we have to write code that is exactly "correct" according to the Python interpreter, otherwise our code won't run. Replacement grammar is when you are able to describe an infinitely large language with a small set of rules.

Python Expressions

A Python "expression" is a legal Python statement. For example: print 1 + 1 is a valid expression, but print 1 + (without a number at the end) is not.

Input -> Function-> Output

What is a Function?

A function takes an input, does something with it, and then produces an output. Functions are convenient because they can help you divide your code into apporpriate blocks, by condensing complicated problems into simpler ones. Some other advantages are that you can avoid repetition of code, as well as reusing code.

In other words, functions are little self-contained programs that perform a specific task. After you have created a function, you can use it at any time, in any place. This saves you the time and effort of having to retell the computer what to do every time it does a common task.

How to Use a Function
A function is made by using the def command and then given a name followed by the function parameters in opening and closing parentheses with any input value. These parameters will eventually be replaced by actual values when the function is used.

Control Flow & Loops: If & While

Making Decisions with Code

The operators for making decisions in Python give a boolean (true or false) result.
Ex: < number >< operator >< number >
print 2<3
The operators used are greater than (>), less than (<), equal to (==) and not equal to (!=). The double equals sign is used because the single equal symbol is already used for assignment variables.

The IF Function
To use the IF function, type if < expression >: then under that indent and type which will define what code is run if the expressions is true. If the expression is false it will skip this code.

You can also define an ELSE statement using the IF function; meaning if the IF function runs a result which is false it will then go to the ELSE statement and run that.

Evaluating OR
< expression >or< expression>
If the first expression evaluates to true, the value is true and the second expression is not evaluated. If the first expression evaluates to false, the value is the value of the second expression.
Loops
A loop is a way of repeating a certain action. The WHILE loop starts with an expression that will validate the running of the loop followed by a colon. This is called a test expression. Inside of that we have the block that will run as long as the test expression is true. The difference between this and the IF command is that once it runs the command it will go back to check if the test expression is still true. If it is, it will run the block of code again, and it will keep doing that until the test expression shows a false value.
Ex:
i = 0
while i < 10:
print i
i = i + 1
This will result in the code printing from 0 to 9. Once it gets to 10 it is no longer less than 10 so the code will not work. An Infinite Loop is one that never finds a false value so will keep running.
Break statement
A break gives you a way to stop the loop even if the condition is still true. You could stick this in an IF command in the WHILE loop which will check to see if there is a reason to break the process or let it continue. If this IF statement is true and you break out of the loop you will then continue the code after the while loop. If it the IF statement is false it will continue to run the code and the loop and will check the IF statement again next time around.
Debugging
A bug is an error in a code which makes it crash. The traceback feature tells you what line it crashed on, what file it was running and how it got there.

Object oriented programming (OOP)

Object Oriented Programming
Classes and Object Oriented Programming (OOP), are techniques used by software engineers on a daily basis. They can use these techniques to write and share code amongst eachother. OOP is a valuable tool for large programing projects because it provides a modular structure for programs which makes it good for defining abstract datatypes where implementation details are hidden. OOP also makes it easy to maintain and modify existing code, as well as providing a good framework for code libraries.
Abstraction
It is a way of representing essential features without including the background details or explanations. Basically, the hiding of details behind documentation.
OOP concepts
There are four main Object Oriented Programming (OOP) Concepts.
1. Encapsulation
2. Abstraction
3. Inheritance
4. Polymorphism.
Object oriented vs. Procedural
One of the important characteristics of procedural programming is that it relies on procedures that operate on data which are two separate concepts. In object-oriented programming, these two concepts are bundled into objects. This makes it possible to create more complicated behavior with less code. Once you have created an object with more complex behavior, you can use it anywhere in your code.

Important OOP definitions

Classes
A class is a collection of objects of similar type. Once a class is defined, any number of objects can be created which belong to that class. It is like a blueprint of a building. It knows the height of the building, number of rooms in the building. The same blueprint can build multiple different buildings (instance/object).
Objects
Objects are the basic run-time entities in an object-oriented system. Programming problem is analyzed in terms of objects and nature of communication between them. When a program is executed, objects interact with each other by sending messages. Different objects can also interact with each other without knowing the details of their data or code.
Inheritance
Inheritance is the process by which objects can acquire the properties of objects of other class. In OOP, inheritance provides reusability, like, adding additional features to an existing class without modifying it. This is achieved by deriving a new class from the existing one. The new class will have combined features of both the classes.
Instance
Multiple different instances and object can be created by one class, because a class is like a blue print. A class is a blueprint and its objects are examples or instances of that blueprint, ex: brad.
Library
A library is a collection of resources used by computer programs to develop software.
Method
A method in OOP is a procedure associated with an object class. An object is made up of behavior and data. Data is represented as properties of the object and behavior as methods.
Program Modules
Software development requires that large programs are divided into smaller program modules to be manageable.
